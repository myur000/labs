## 2. Краткие теоретические сведения

**Терминал** — это программа, которая позволяет пользователю взаимодействовать с операционной системой при помощи командной строки. Через терминал можно запускать программы, просматривать файлы, управлять каталогами, получать информацию о системе, создавать пользователей и выполнять административные действия.

В Linux командная строка обычно содержит имя пользователя, имя компьютера и текущий каталог. Например:

```bash
student@ubuntu:~$
```

Здесь:

- `student` — имя текущего пользователя;
- `ubuntu` — имя компьютера;
- `~` — домашний каталог пользователя;
- `$` — признак обычного пользователя;
- `#` — признак суперпользователя root.

Многие команды Linux при успешном выполнении не выводят сообщений. Это соответствует принципу: **«молчит — значит работает»**.

Для выполнения административных команд используется `sudo`. Например:

```bash
sudo adduser testuser
```

После ввода такой команды система попросит пароль текущего пользователя. При вводе пароль не отображается на экране, что является нормальным поведением Linux.

## 3. Основные системные каталоги Linux

| Каталог | Назначение |
|---|---|
| `/bin` | Основные пользовательские программы и команды. |
| `/sbin` | Системные команды для администрирования. |
| `/boot` | Файлы, необходимые для загрузки системы. |
| `/home` | Домашние каталоги пользователей. |
| `/dev` | Файлы устройств. |
| `/etc` | Конфигурационные файлы системы и программ. |
| `/lib` | Системные библиотеки и модули ядра. |
| `/media` | Точки подключения съёмных носителей. |
| `/mnt` | Временные точки монтирования файловых систем. |
| `/opt` | Дополнительные программные пакеты. |
| `/proc` | Виртуальная файловая система с информацией о ядре и процессах. |
| `/root` | Домашний каталог суперпользователя root. |
| `/tmp` | Временные файлы. |
| `/var` | Часто изменяемые данные: журналы, очереди, кэш. |
| `/usr` | Пользовательские программы, библиотеки, документация. |

## 4. Ход работы

### 4.1. Запуск Linux Ubuntu и терминала

Для выполнения лабораторной работы была запущена операционная система Linux Ubuntu. После входа в систему был открыт терминал.

Открыть терминал можно несколькими способами:

```text
Applications → Accessories → Terminal
```

или сочетанием клавиш:

```text
Ctrl + Alt + T
```

После запуска терминала появилась командная строка:

```bash
student@ubuntu:~$
```

Это означает, что пользователь `student` находится в домашнем каталоге.

---

### 4.2. Получение справки по командам

Для получения справки о команде используется параметр `--help`.

Пример:

```bash
date --help
```

Пример результата:

```text
Usage: date [OPTION]... [+FORMAT]
Display date and time in the given FORMAT.
```

Вывод: с помощью `--help` можно узнать назначение команды и доступные параметры.

## 5. Выполнение основных команд

### 5.1. Команда `date`

Команда `date` показывает текущую дату и время системы.

```bash
date
```

Пример результата:

```text
Пт 24 апр 2026 14:30:15 CEST
```

Вывод: команда отображает текущие дату, время и часовой пояс.

### 5.2. Команда `oclock`

Команда `oclock` запускает графические аналоговые часы.

```bash

```


## 6. Создание нового пользователя

Для создания нового пользователя была использована команда `adduser`.

```bash
sudo adduser labuser
```

Пример процесса выполнения:

```text
Adding user `labuser' ...
Adding new group `labuser' (1001) ...
Adding new user `labuser' (1001) with group `labuser' ...
Creating home directory `/home/labuser' ...
Copying files from `/etc/skel' ...
New password:
Retype new password:
passwd: password updated successfully
```

После создания пользователя был задан пароль.

Для добавления пользователя в административную группу была выполнена команда:

```bash
sudo usermod -aG sudo labuser
```

В некоторых старых версиях Ubuntu использовалась группа `admin`. Если такая группа есть в системе, можно использовать команду:

```bash
sudo usermod -aG admin labuser
```

Проверка групп пользователя:

```bash
groups labuser
```

Пример результата:

```text
labuser : labuser sudo
```

Вход под новым пользователем:

```bash
su - labuser
```

Пример результата:

```bash
labuser@ubuntu:~$
```

Проверка текущего пользователя:

```bash
whoami
```

Пример результата:

```text
labuser
```

Вывод: новый пользователь `labuser` был создан, ему был задан пароль, после чего он был добавлен в административную группу и проверен вход в систему.

## 9. Типы файлов в Linux

В Linux файл рассматривается как последовательность байтов. При этом через файловую систему представлены не только обычные документы, но и устройства, каналы и другие объекты.

Основные типы файлов в Linux:

| Тип файла | Описание |
|---|---|
| Обычные файлы | Текстовые документы, изображения, программы, библиотеки. |
| Каталоги | Содержат ссылки на другие файлы и каталоги. |
| Файлы устройств | Представляют устройства компьютера, например диски или терминалы. |
| Блочные устройства | Устройства, работающие с блоками данных, например жёсткие диски. |
| Символьные устройства | Устройства, работающие с потоком символов, например клавиатура. |
| Символические ссылки | Ссылки на другие файлы или каталоги. |
| Именованные каналы | Используются для обмена данными между процессами. |
| Сокеты | Используются для сетевого и межпроцессного взаимодействия. |

## 10. Контрольные вопросы

### 1. Что такое терминал?

Терминал — это программа для работы с командной строкой Linux. С его помощью пользователь вводит команды и получает результат их выполнения. Через терминал можно управлять файлами, пользователями, процессами, настройками системы и программами.

---

### 2. Перечислите основные системные каталоги Linux.

К основным системным каталогам Linux относятся:

- `/bin` — основные команды;
- `/sbin` — системные административные команды;
- `/boot` — файлы загрузки системы;
- `/home` — домашние каталоги пользователей;
- `/dev` — файлы устройств;
- `/etc` — конфигурационные файлы;
- `/lib` — библиотеки;
- `/media` — точки подключения съёмных носителей;
- `/mnt` — временные точки монтирования;
- `/opt` — дополнительные программы;
- `/proc` — информация о процессах и ядре;
- `/root` — домашний каталог root;
- `/tmp` — временные файлы;
- `/var` — изменяемые данные и журналы;
- `/usr` — программы, библиотеки и документация.

---

### 3. Расскажите о типах файлов в ОС Linux.

В Linux существуют обычные файлы, каталоги, файлы устройств, символические ссылки, именованные каналы и сокеты. Обычные файлы используются для хранения данных, каталоги — для организации файловой структуры, файлы устройств — для взаимодействия с оборудованием, символические ссылки — для создания ссылок на другие объекты, а каналы и сокеты — для обмена данными между процессами.
